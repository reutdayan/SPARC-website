Leveraging Graph Geometry for Cold-Start Node Prediction

SPARC learns a feature-to-spectral map from the observed training graph, then retrieves pseudo-neighborhoods for cold-start nodes from features alone—without test-time edges.

Reut Dayan and Uri Shaham
Paper Code Video BibTeX

Abstract

A geometry-aware view of cold-start node prediction

Cold-start nodes pose a fundamental challenge for node prediction in graphs: at inference time, newly arriving nodes have no observed edges. This breaks a core assumption of many graph neural networks, which rely on observed adjacency to aggregate neighborhood information.

SPARC addresses this problem by learning a parametric map from node features to approximate spectral coordinates of the observed training graph. At inference time, a cold-start node is embedded from its features alone, and its pseudo-neighborhood is retrieved in the learned spectral space.

01

Strict cold-start setting

Many graph learning models assume that the target node has an observed neighborhood at inference time. This assumption breaks in strict cold-start prediction, where a newly arriving node is unseen during training and has no incident edges when the prediction must be made.

In this setting, the node feature vector is available, but the graph structure around the node is hidden. Therefore, the model cannot directly use standard message passing or neighborhood aggregation for the cold-start node.

02

Pseudo-neighborhood retrieval

We can approximate the neighbors using the equation below. The function ϕ defines the representation space, and retrieval is performed by finding the nearest training nodes to the cold-start node in that space.

N̂ϕk(vcold) = kNN ( ϕ(vcold) ; {ϕ(u)}u ∈ Vtr )

This makes the retrieval space a central design choice. Once a pseudo-neighborhood is retrieved, an existing downstream graph model can use it as contextual information for prediction.

03

Why feature retrieval fails

A simple baseline is to retrieve pseudo-neighbors directly in the raw feature space. This is attractive because it is fully inductive: every cold-start node has a feature vector, so retrieval can always be performed.

However, feature similarity can be a weak proxy for graph connectivity. Two nodes may be close in the graph because of relational behavior, interactions, citations, or shared users, even if their raw attributes are different.

The Reddit example illustrates this limitation: posts are represented by text features, while graph edges depend on user-comment interactions. A user may comment on posts about different topics, so connected posts may not be close in feature space.

04

Spectral graph geometry

Graph Laplacian eigenvectors provide coordinates shaped by connectivity.

Spectral embeddings come from Laplacian eigenvectors, so they reflect connectivity rather than raw attributes.

Structurally close nodes tend to be nearby in spectral space—ideal for pseudo-neighborhood retrieval.

Exact spectral coordinates are unavailable for cold-start nodes, whose test-time edges are hidden.

05

The SPARC idea

Learn an inductive bridge from node attributes to graph-aware geometry.

SPARC bridges the gap between feature-only inference and spectral graph geometry. It learns a parametric encoder Fθ that maps node features into spectral-like coordinates.

During training, SPARC uses the observed graph to learn a representation that approximates the graph's spectral geometry. During inference, the learned encoder can embed a cold-start node using only its features.

This allows the model to place a new node into a graph-aware retrieval space without observing its test-time edges and without recomputing graph eigenvectors.

06

Training objective

The encoder is optimized to preserve graph geometry and optionally align with labels.

The SPARC encoder is trained on the observed training graph. The spectral objective encourages embeddings to be smooth with respect to the graph: structurally related nodes should have compatible representations.

LSPARC = Lspec + λQLcon = 1m Tr( ZB⊤ (LB − λQQB) ZB )

An orthogonality constraint prevents the embeddings from collapsing to a trivial solution. When labels are available, SPARC can also add a task-aware constraint that pulls same-label nodes closer and pushes different-label nodes farther apart.

07

Cold-start inference

At inference time, a cold-start node arrives with only its feature vector. SPARC first applies the trained encoder to obtain the node's spectral-like representation.

Then, it retrieves the nearest observed training nodes in the learned SPARC space. These retrieved nodes form the pseudo-neighborhood that supplies structural context to the downstream model.

The downstream predictor receives the cold-start node and its retrieved context, allowing the model to behave more like a graph model even though the target node has no observed edges.

08

Plug-in retrieval module

SPARC is designed as a retrieval module rather than a new graph classifier. It replaces the unavailable cold-start neighborhood with a learned pseudo-neighborhood.

Because of this separation, the same retrieval idea can be used with different graph backbones. The paper evaluates SPARC with GraphSAGE, Spectral-GCN, and NAGphormer.

This makes the method modular: existing graph architectures can be adapted to strict cold-start settings by changing how their neighborhood context is constructed.

Results

SPARC improves retrieval and cold-start prediction

The results evaluate whether SPARC recovers better pseudo-neighborhoods and whether this improved retrieval translates into stronger strict cold-start classification.

Across the reported datasets, SPARC improves over raw feature retrieval and Cold-Brew in the main classification benchmark, while the neighborhood recovery table shows that learned spectral retrieval provides a substantially stronger proxy for graph context.
